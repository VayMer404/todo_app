# Todo App
Веб-приложение для управления задачами
## Описание проекта
в приложение реализованно:
* Регистрирация аккаунта
* вход в систему
* Выход из аккаунта
* Создание задач
* Редактирование задачи
* Удаление задачи
* Отмечать задачи как выполненные
* Просматривать свои задачи отмеченные в профиле

(Для авторизации используется JWT-токен)
# Используемый стек технологий

## Frontend

* Next.js 16
* React
* TypeScript

## Backend

* Next.js API Routes
* Prisma ORM v6.19.3

## База данных

* SQLite

## Авторизация

* JSON Web Token (JWT)
* bcrypt

# Установленные зависимости

Основные пакеты:

```bash
npm install prisma@6.19.3
npm install bcrypt
npm install jsonwebtoken
```

Для TypeScript:

```bash
npm install -D @types/jsonwebtoken
npm install -D @types/bcrypt
```

# Структура базы данных

## User
Пользователь содержит:

* id
* email
* password

## Task
Задача содержит:

* id
* title
* description
* status
* createdAt
* updatedAt
* userId

Каждая задача принадлежит определённому пользователю через поле userId.

# Настройка проекта

## Клонирование репозитория

```bash
git clone <URL_РЕПОЗИТОРИЯ>
```

Переход в папку проекта:

```bash
cd todo-app
```


# Установка зависимостей

```bash
npm install
```


# Настройка переменных окружения

Создать файл `.env` на основе `.env.example`.

Пример:

```env
DATABASE_URL="file:./dev.db"
JWT_SECRET="your-secret-key"
```

# Создание базы данных

Выполнить миграцию Prisma:

```bash
npx prisma migrate dev
```

После выполнения команды будет создана база данных SQLite.

# Запуск проекта

Запуск сервера разработки:

```bash
npm run dev
```

После запуска приложение будет доступно по адресу:

```txt
http://localhost:3000
```
Чтобы переходить по страницам можно в конце написать на какую страницу вы хотите перейти. К примеру http://localhost:3000/register - страница регистрации;
http://localhost:3000/login - вход в профиль; http://localhost:3000/dashboard - главная страница сервиса;

# Реализованный функционал

### Регистрация

Пользователь может создать аккаунт с уникальным email.

Пароль проходит проверку на минимальную длину (не менее 6 символов).

Пароль хранится в базе данных в зашифрованном виде с использованием bcrypt.

### Авторизация

После успешного входа создаётся JWT-токен.

JWT сохраняется в cookies браузера.

### Создание задач

Авторизованный пользователь может создавать задачи.

### Редактирование задач

Пользователь может изменить название задачи.

### Изменение статуса

Задача может быть отмечена как выполненная или невыполненная.

### Удаление задач

Пользователь может удалить задачу.

### Изоляция данных пользователей

Каждый пользователь видит только свои задачи.
